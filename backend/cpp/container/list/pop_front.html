If empty() is true, the behavior is undefined. (until C++26) If empty() is true: If the implementation is hardened, a contract violation occurs. Moreover, if the contract-violation handler returns under “observe” evaluation semantic, the behavior is undefined. If the implementation is not hardened, the behavior is undefined. (since C++26) References and iterators to the erased element are invalidated. [edit] Complexity Constant.
