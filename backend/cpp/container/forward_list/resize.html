If the current size is greater than count, the container is reduced to its first count elements. If the current size is less than count, then: 1) Additional default-inserted elements are appended. 2) Additional copies of value are appended. [edit] Complexity Linear in the difference between the current size and count. Additional complexity possible due to list traversal to reach the first element to erase/the end position to insert. Notes If value-initialization in overload (1) is undesirable, for example, if the elements are of non-class type and zeroing out is not needed, it can be avoided by providing a custom Allocator::construct.
