[edit] Notes The underlying iterator of the returned reverse iterator is the end iterator. Hence the returned iterator is invalidated if and when the end iterator is invalidated. [edit] Example Run this code #include #include #include #include   int main() { const std::flat_map coins { {10, "dime"}, {100, "dollar"}, {50, "half dollar"}, {5, "nickel"}, {1, "penny"}, {25, "quarter"} }; // initializer entries in name alphabetical order   std::cout << "US coins in circulation, largest to smallest denomination:\n"; for (auto it = coins.crbegin(); it != coins.crend(); ++it) std::cout << std::setw(11) << it->second << " = ¢" << it->first << '\n'; } Output: US coins in circulation, largest to smallest denomination: dollar = ¢100 half dollar = ¢50 quarter = ¢25 dime = ¢10 nickel = ¢5 penny = ¢1 [edit] See also rendcrend returns a reverse iterator to the end (public member function) [edit] rbegincrbegin(C++14) returns a reverse iterator to the beginning of a container or array (function template) [edit]
