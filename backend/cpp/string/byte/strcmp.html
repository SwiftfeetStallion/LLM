The sign of the result is the sign of the difference between the values of the first pair of characters (both interpreted as unsigned char) that differ in the strings being compared. The behavior is undefined if lhs or rhs are not pointers to null-terminated strings. Contents 1 Parameters 2 Return value 3 Example 4 See also [edit] Parameters lhs, rhs - pointers to the null-terminated byte strings to compare [edit] Return value Negative value if lhs appears before rhs in lexicographical order. Zero if lhs and rhs compare equal. Positive value if lhs appears after rhs in lexicographical order.
