It is an error if p does not refer to a symbolic link. The non-throwing overload returns an empty path on errors. Contents 1 Parameters 2 Return value 3 Exceptions 4 Example 5 See also [edit] Parameters p - path to a symlink ec - out-parameter for error reporting in the non-throwing overload [edit] Return value The target of the symlink (which may not necessarily exist). [edit] Exceptions Any overload not marked noexcept may throw std::bad_alloc if memory allocation fails. 1) Throws std::filesystem::filesystem_error on underlying OS API errors, constructed with p as the first path argument and the OS error code as the error code argument. 2) Sets a std::error_code& parameter to the OS API error code if an OS API call fails, and executes ec.clear() if no errors occur.
