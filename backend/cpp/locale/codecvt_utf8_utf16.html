If Elem is a 32-bit type, one UTF-16 code unit will be stored in each 32-bit character of the output sequence. This is an N:M conversion facet, and cannot be used with std::basic_filebuf (which only permits 1:N conversions, such as UTF-32/UTF-8, between the internal and the external encodings). This facet can be used with std::wstring_convert.
