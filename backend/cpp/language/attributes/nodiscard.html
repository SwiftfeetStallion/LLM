Contents 1 Syntax 2 Explanation 3 Example 4 Standard library 4.1 Allocation functions 4.2 Indirect access 4.3 Emptiness-checking functions 4.4 Miscellaneous 5 Defect reports 6 References 7 See also [edit] Syntax [[nodiscard]] (1) (since C++17) [[nodiscard( string-literal )]] (2) (since C++20) string-literal - an unevaluated string literal that could be used to explain the rationale for why the result should not be discarded [edit] Explanation Appears in a function declaration, enumeration declaration, or class declaration. If, from a discarded-value expression other than a cast to void, a function declared nodiscard is called, or a function returning an enumeration or class declared nodiscard by value is called, or a constructor declared nodiscard is called by explicit type conversion or static_cast, or an object of an enumeration or class type declared nodiscard is initialized by explicit type conversion or static_cast, the compiler is encouraged to issue a warning. The string-literal, if specified, is usually included in the warnings.
