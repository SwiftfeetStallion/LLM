The value of that object must be previously obtained by a call to std::feholdexcept or std::fegetenv or be a floating-point macro constant. If any of the floating-point status flags are set in envp, they become set in the environment (and are then testable with std::fetestexcept), but the corresponding floating-point exceptions are not raised (execution continues uninterrupted) [edit] Parameters envp - pointer to the object of type std::fenv_t which holds the status of the floating-point environment [edit] Return value ​0​ on success, non-zero otherwise.
