Contents 1 Parameters 2 Return value 3 Remarks 4 See also [edit] Parameters desired - the weak_ptr to store into the atomic variable order - memory order constraints to enforce [edit] Return value (none) [edit] Remarks All changes to the atomic_weak_ptr object itself, and all associated use_count increments, are guaranteed to be performed atomically. Associated use_count decrements take place after the atomic operation, but are not required to be part of it. Any associated destruction or deallocation operations take place after the atomic operation and are not part of it.
