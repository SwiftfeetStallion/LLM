noexcept(noexcept(ranges::iter_move(std::declval()))         requires std::input_iterator; (since C++20) Casts the result of dereferencing the underlying iterator to its associated rvalue reference type. The function body is equivalent to: return std::ranges::iter_move(std::get(i.var));. This function is not visible to ordinary unqualified or qualified lookup, and can only be found by argument-dependent lookup when std::common_iterator is an associated class of the arguments. If i.var does not hold an I object (i.e. an iterator), the behavior is undefined.
